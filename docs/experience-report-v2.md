## 二、深度体验发现

### 2.1 返程清点体验（新发现）

**现状分析：**
```javascript
// 返程模式判断逻辑（清晰）
const isReturn = computed(() => mode.value === 'return')

// 消耗品返程不可勾选（合理）
function isTrackableItem(item) {
  if (isReturn.value && item.isConsumable) return false
  return true
}
```

**问题：**
1. **入口不明显**：用户不知道可以切换到返程清点模式
2. **缺少对比视图**：无法直观看到出发时带了什么、返程时应带回什么
3. **需带回物品无特殊标记**：重要物品（证件、电子设备）在返程时应该高亮

**建议优化：**
- 清单页顶部badge增加「切换模式」按钮
- 返程模式下，需带回物品增加「带回」标签
- 增加「遗漏检查」功能，提示用户哪些物品还未勾选

### 2.2 装备库与清单的割裂（核心痛点）

**现状：**
- 装备库：管理个人物品（add/remove/updateItem）
- 清单：基于模板创建，独立管理物品
- **两者数据完全独立**

**使用场景断裂：**
```
用户场景：
1. 在装备库添加了「索尼相机」
2. 创建行程时从模板导入
3. 模板里的「相机」是通用项，不是用户特定的「索尼相机」
4. 用户需要手动修改或重新添加
```

**建议优化方案：**
1. **装备库物品直接加入清单**
   - 清单页添加物品时，优先显示装备库
   - 支持从装备库多选快速添加

2. **智能匹配**
   - 模板导入时，自动匹配装备库中的相似物品
   - 例如：模板中的「相机」→ 匹配装备库的「索尼相机」

3. **常用物品自动推荐**
   - 基于历史行程，自动推荐常用装备

### 2.3 模板选择流程仍可优化

**现状流程：**
```
首页 → 选择模板 → 填写标题/日期/目的地 → 确认创建 → 进入清单
```

**问题：**
1. 步骤过多，快速使用场景下显得繁琐
2. 模板预览不足，用户不确定模板内容
3. 没有「一键使用」快速创建

**建议优化：**
- **快速创建模式**：首页增加「+」按钮，点击后弹窗选择模板，自动使用默认信息创建
- **模板预览**：模板卡片支持展开预览物品列表
- **智能推荐**：根据日期、天气、历史记录推荐模板

### 2.4 数据备份机制的隐患

**现状：**
```javascript
// 备份到剪贴板
function doExport() {
  const data = exportAll()
  uni.setClipboardData({ data: JSON.stringify(data) })
}
```

**问题：**
1. 剪贴板数据易被覆盖
2. 大数据量时剪贴板可能截断
3. 无自动备份机制
4. 恢复操作复杂，需要用户手动粘贴

**建议优化：**
- **本地自动备份**：每次修改后自动备份到本地存储（保留最近10个版本）
- **导出到文件**：支持导出到微信文件助手或本地文件
- **云同步优化**：一键开启自动云同步

### 2.5 历史记录功能的价值未被充分利用

**亮点发现：**
```javascript
// 找回率统计（独特功能）
const avgRecoveryRate = computed(() => {
  const rates = archived.value
    .filter((item) => typeof item.recoveryRate === 'number')
    .map((item) => item.recoveryRate)
  // ...
})
```

**问题：**
1. 用户不清楚「找回率」的含义
2. 缺少历史行程的数据分析（如：最常带哪些物品、遗漏率最高的物品）
3. 批量管理功能隐藏较深

**建议优化：**
- 空状态时显示「找回率说明」引导
- 增加「旅行档案」功能，统计用户出行习惯
- 历史详情页显示该行程的「应带/实带/遗漏」分析

## 四、技术实现建议

### 4.1 装备库与清单打通的数据结构

```javascript
// 装备库物品增加「常用」标记
{
  id: 'gear_001',
  name: '索尼相机',
  category: '电子',
  isFrequent: true,  // 常用标记
  lastUsed: '2024-06-15',  // 上次使用时间
  useCount: 5  // 使用次数
}

// 清单添加物品时智能匹配
function addItemToTrip(itemName, tripId) {
  // 1. 查找装备库匹配
  const matchedGear = gearStore.items.find(g => 
    g.name.includes(itemName) || itemName.includes(g.name)
  )
  
  // 2. 如果匹配，使用装备库详细信息
  if (matchedGear) {
    return tripStore.addCustomItem({
      ...matchedGear,
      gearId: matchedGear.id  // 关联装备库ID
    })
  }
  
  // 3. 无匹配，使用默认信息
  return tripStore.addCustomItem({ name: itemName })
}
```

### 4.2 自动备份机制

```javascript
// 每次修改后自动备份
function autoBackup() {
  const backup = {
    timestamp: Date.now(),
    trips: tripStore.exportData(),
    templates: templateStore.exportData(),
    gear: gearStore.exportData()
  }
  
  // 获取现有备份
  let backups = uni.getStorageSync('rtg_auto_backups') || []
  
  // 保留最近10个版本
  backups.unshift(backup)
  if (backups.length > 10) backups = backups.slice(0, 10)
  
  uni.setStorageSync('rtg_auto_backups', backups)
}

// 监听所有store变化
watch(() => tripStore.trips, autoBackup, { deep: true })
watch(() => templateStore.templates, autoBackup, { deep: true })
watch(() => gearStore.items, autoBackup, { deep: true })
```
