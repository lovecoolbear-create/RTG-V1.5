# ReadyToGo 首页行程驱动 PDR

## 1. 文档目标

定义首页行程系统的核心业务流程、状态机与交互驱动规则，统一“新建日程”“快速出发”“返程清点”“归档迁移”逻辑，作为后续产品、设计与开发的共同基线。

## 2. 范围

- 首页操作区中的两个入口：新建日程、快速出发
- 日历日期驱动的当日行程展示
- 行程卡片状态变化与迁移
- 清单流程（出发与返程）
- 模板清单实例化、编辑与回存模板库

不含：

- 账号体系与云端多端同步
- 推送通道的实际发送策略
- 历史详情页的展示设计

## 3. 核心对象定义

### 3.1 行程 Trip

字段建议：

- id
- title
- source: planned | temporary
- templateId
- date
- status: preparation | packing | packed | departed | returnPhase
- progress: 0~1
- checklistSnapshot
- alarmEnabled
- alarmTime

### 3.2 模板 Template

- id
- name
- items[]
- updatedAt

### 3.3 清单实例 Checklist Session

- tripId
- mode: departure | return
- items[]
- edited: boolean

## 4. 状态机定义

### 4.1 出发阶段

- preparation：已创建行程，未开始清点
- packing：进入清单流程，清点进行中
- packed：清点完成，选择“稍后出发”
- departed：清点完成，选择“现在出发”

### 4.2 返程阶段

- returnPhase：返程清点中或返程完成待迁移
- 返程清点完成后，行程从“计划中的行程”迁移到“待归档的行程”

## 5. 交互路径

### 5.1 路径 A：计划出行

1. 点击“新建日程”
2. 填写：行程名称、模板、出行日期，确认
3. 用户在日历点击对应日期，操作区展示当日行程卡片
4. 点击卡片，进入模板清单流程（自动带入已填写的名称与日期）
5. 清点完成，弹窗二选一：
   - 现在出发：状态变为 departed
   - 稍后出发：状态变为 packed
6. 返回操作区显示最新状态

### 5.2 路径 B：临时出发

1. 点击“快速出发”
2. 弹出模板选择与行程信息填写（名称、模板、出行日期）
3. 进入清单流程
4. 清点完成，弹窗二选一：
   - 现在出发：状态变为 departed
   - 稍后出发：状态变为 packed
5. 返回操作区显示最新状态

### 5.3 返程触发（A/B 共用）

1. 点击状态为 departed 的行程卡片
2. 进入返程清点模板（默认复用出发模板）
3. 清单完成，弹窗“行程结束”
4. 卡片从“计划中的行程”迁移到“待归档的行程”

## 6. 模板与清单规则

### 6.1 模板实例化

- 每次进入清单流程时，从模板库拉取模板并创建 checklistSnapshot
- 清点中编辑的是当前行程快照，不直接修改模板本体

### 6.2 清点中编辑能力

- 支持新增条目
- 支持删除条目
- 支持修改条目

### 6.3 回存模板库

清点结束后支持三种动作：

- 不保存到模板库
- 另存为新模板
- 覆盖原模板

覆盖原模板需二次确认。

## 7. 页面驱动规则

- 日历日期是“当日行程列表”的过滤条件
- 操作区展示“所选日期”对应行程
- 行程卡片点击行为由状态决定：
  - preparation/packing/packed：进入出发清点或出发确认流程
  - departed：进入返程清点流程
  - returnPhase：进入返程流程收尾或归档确认

## 8. 异常与边界

- 未选择模板时禁止进入清单流程
- 模板条目为空时允许进入清单并手动补录
- 清点中断后再次进入，应恢复到上次 checklistSnapshot
- 同一天多行程时按创建时间倒序展示

## 9. 验收标准

- 新建日程可完整走通到 packed/departed
- 快速出发可完整走通到 packed/departed
- departed 行程可触发返程并迁移至待归档
- 清单编辑后可选择回存模板库，且不误改原模板
- 首页卡片状态、列表归属与清单阶段保持一致
